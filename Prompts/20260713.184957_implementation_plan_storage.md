# User Prompt
```
is it possible to extract game data/level into text/json files and store it outside of project source code.
The app should be able to read the text/json files at run time and load the game.
in settings, provide a text field to save the game level location.
The dafault game levels path is '/sdcard/Vypeensoft/Flow_Free/game/levels/'
Also save all settings in a json file at '/sdcard/Vypeensoft/Flow_Free/settings/settings.json'
```

# Storage Implementation Plan

This plan outlines the steps to extract settings and game level loading logic from internal app components/assets to the external SD card, fulfilling the user's requirements.

## Proposed Changes

### UI & Settings Initialization (`MainActivity.kt`)
- Request `MANAGE_EXTERNAL_STORAGE` permission on startup for Android 11+ to gain broad file access to the `/sdcard/` root directory.
- Update `SettingsScreen` UI to display a new text field for the user to configure the `levelsPath`.

### Settings I/O (`SettingsManager.kt`)
- Create a new `SettingsManager` object utilizing `kotlinx.serialization.json`.
- Expose a `StateFlow` for `AppSettings`, reading and writing directly to `/sdcard/Vypeensoft/Flow_Free/settings/settings.json`.
- Migrate `PipesViewModel` dependencies away from Android DataStore over to `SettingsManager`.

### Dynamic Level Loading (`PipeModels.kt`)
- Refactor the `LevelPack.init()` logic.
- Instead of unconditionally parsing from `context.assets`, it will check if the user-defined `levelsPath` directory exists and has `.json` files.
- If it doesn't, it will default to the assets folder, but additionally, it will extract and save those assets to the SD card directory to seed the initial custom levels.
