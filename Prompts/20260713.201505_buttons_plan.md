# Consistent Buttons Implementation Plan

This plan outlines how we will keep the Undo, Restart, and Solution buttons always visible and adjust their enabled states correctly.

## Proposed Changes

### `MainActivity.kt`
- Make all 3 buttons (Undo, Restart, Solution) always present in the `actionButtons` block.
- **Undo Button Enable Condition**: Enabled when `history` is not empty.
- **Restart Button Enable Condition**: Enabled when `history` is not empty OR `isLevelWon` is true (to allow clearing a finished/shown solution).
- **Solution Button Enable Condition**: Enabled when a solution file exists (`currentLevelStats != null`).

### `PipesViewModel.kt`
- **`onRestart()`**: Remove the strict `isLevelWon` and `history.isEmpty()` block so users can restart even after the level is won or a solution is loaded. It will now just reset everything back to a clean slate.
- **`onUndo()`**: Remove the `isLevelWon` block. Additionally, when a user undos a move from a "won" state, we must ensure we reset `isLevelWon = false` so they can continue playing from that previous step.
- **`loadSolution()`**: As the solution is animated cell-by-cell, we will push each step to the `history` list. This gives the user the ability to actually "Undo" the solution playback step-by-step!

## Verification Plan
1. Open a new level. The 3 buttons will be visible. Solution will be disabled (if not beaten yet), and Undo/Restart will be disabled.
2. Make a move. Undo/Restart become enabled.
3. Open a beaten level. Click Solution. The animation plays. Afterward, Restart and Undo are enabled. You can click Undo to rewind the solution, or Restart to clear it.
