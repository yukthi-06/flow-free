# User Prompt
```
when the user plays the game, is it possible to capture the moves and store the solution in a file(json or any industry standard)
```

# Solution Capturing Implementation Plan

This plan outlines the steps to automatically capture and save the user's solution when they win a level.

## Proposed Changes

### Data Models (`PipeModels.kt`)
We need to define a standardized JSON structure for saving solutions.
- Add the `@Serializable` annotation to the existing `CellPos` class so it can be easily converted to JSON.
- Create two new serializable models for the solution data:
  ```kotlin
  @Serializable
  data class PathData(val colorIndex: Int, val path: List<CellPos>)

  @Serializable
  data class SolutionData(
      val levelId: String,
      val optimalMoves: Int,
      val playerMoves: Int,
      val solution: List<PathData>
  )
  ```

### Game Logic (`PipesViewModel.kt`)
We will hook into the moment the player successfully finishes a level to serialize the data.
- In the `onLevelWon()` function, capture the final state of all drawn paths (`gameState.paths`).
- Map these paths into the new `SolutionData` object.
- Create the directory `/sdcard/Vypeensoft/Flow_Free/solutions/` if it does not exist.
- Write the serialized JSON string to a file named `<level_id>_solution.json` (e.g., `5x5_001_solution.json`) within an IO coroutine block.
