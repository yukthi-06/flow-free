# User Prompt
```
Add a button at the bottom for completed games.  The label should be 'Show Solution'.  Clicking on this should read the solution file and draw the solution on the screen
```

# Show Solution Feature Implementation Plan

This plan outlines how we will add the requested "Show Solution" button for previously completed games.

## Proposed Changes

### `PipesViewModel.kt`
- Add a new function `loadSolution()` to the ViewModel.
- This function will:
  1. Determine the current level's ID.
  2. Attempt to read `/sdcard/Vypeensoft/Flow_Free/solutions/<levelId>_solution.json`.
  3. Decode the JSON file back into our `SolutionData` model.
  4. Iterate through the saved paths, reconstruct the `gameState.paths` and `gameState.cellOwner` maps.
  5. Update the UI state with this loaded game state so it renders on the screen, and lock the board by marking `isLevelWon = true`.

### `MainActivity.kt`
- In the `GameScreen` UI component, specifically inside the `actionButtons` block (around line 321), we will add a new `Button`.
- We will check if `currentLevelStats != null` (meaning the player has already beaten this level in the past).
- If true, the "Show Solution" button will be rendered.
- Clicking the button will trigger `viewModel.loadSolution()`.

## Verification Plan
1. Open the app and navigate to a level you have already completed (it should have a star or best score).
2. Look for the "Show Solution" button below the grid or alongside Undo/Restart.
3. Click it to instantly draw the saved solution onto the board.
