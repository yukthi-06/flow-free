# Sync Solutions on Startup Implementation Plan

This plan outlines how the app will check the external SD card for existing solution files on startup and automatically restore the player's level progress if they reinstall the app or clear its data.

## Proposed Changes

### `PipesViewModel.kt`
- Introduce a new private suspend function `syncSolutionsFromDisk()`.
- This function will:
  1. Access the `/sdcard/Vypeensoft/Flow_Free/solutions/` directory.
  2. Scan for any files ending in `_solution.json`.
  3. Parse each file to extract its `levelId` and the `playerMoves`.
  4. Call the existing `repository.updateBestScore(levelId, playerMoves)` to register the level as completed in the app's internal database (SharedPreferences).
  5. Refresh the UI state's `_levelStats` to ensure the UI immediately reflects the restored progress.
- Modify the `init` block of the ViewModel to execute `syncSolutionsFromDisk()` within an IO coroutine at startup, right before `achievementsManager.checkExistingAchievements()`.

## Verification Plan
1. Launch the app and confirm the grid shows your completed levels.
2. Go to Android settings, clear the app's data (simulating a fresh reinstall).
3. Re-launch the app. Because your solution files were persisted on the SD card, the app should instantly detect them and restore all your stars and completion statuses.
